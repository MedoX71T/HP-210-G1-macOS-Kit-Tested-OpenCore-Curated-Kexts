# Contributing

Thanks for your interest in contributing! To keep things organized and fast, please follow these steps:

- Before opening an issue, search existing issues to avoid duplicates.
- Use the **Bug report** template for bugs and **Feature request** template for requests.
- Include an EFI zip and `config.plist` when reporting hardware or boot issues.
- Use `scripts/collect-diagnostics.sh` to gather logs and attach the resulting archive to the issue.

Pull Requests
- Fork this repo, make your change on a branch named `fix/description` or `feat/description`.
- Keep PRs focused and small; one feature/fix per PR.
- Include testing steps and the macOS version used for verification.

Maintainers will review PRs within a few days. For security issues, see `SECURITY.md`.
